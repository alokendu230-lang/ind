HTML MEDIA AND MULTI-PAGE WEBSITE

WHAT IS MULTIMEDIA?

Multimedia is the use of different types of content—like text, images, audio, video, and animations—together in one system.
  • It makes information more interactive and engaging.
  • Examples: Online videos, video games, websites with images and sound, presentations with animation.

In short: Multimedia = multiple types of media combined.

Browser Support
  • The first web browsers could only show plain text, usually in one font and color.
  • Later browsers added support for colors, different fonts, images, and multimedia like audio and video

Multimedia Formats

  • Multimedia elements like audio and video are stored in media files.
  • You can usually tell the type of file by its file extension.
  • Common formats include:
  • Audio: .wav, .mp3
  • Video: .mp4, .mpg, .wmv, .avi

HTML VIDEO

  • HTML allows you to embed videos directly into a webpage using the videotag.
  • You don’t need extra plugins like Flash or QuickTime.
  • HTML video works on all modern browsers and on computers and phones.

    Why we use <source > inside <video > or < audio >
    Because not all browsers support the same file format. Example:

    • Chrome likes .webm
    • Safari (iPhone/Mac) likes.mp4
    If you only give one file, some users may not be able to play it.

    Solution: Use multiple <source >
    • Put several <source >inside <video >
    • Each <source > give
      • src → file location
      • type → file format (so browser knows before downloading).
    • Browser will try sources in order and play the first one it supports.

      Problem:
    • Deaf or hard-of-hearing users can’t follow your video.
    • People in noisy/quiet places can’t listen to the sound.
    • Non-English speakers may not understand the language

    Solution: Use (track)

    The (track) tag adds subtitles/captions to your video. It uses a text file in WebVTT (.vtt) format.

    Key Attributes of :
    1. src → Path of the subtitle file (.vtt).
    2. kind → What type of text?
      • captions → For people who can’t hear (includes sounds).
      • subtitles → Translation into another language.
      • descriptions → For visually impaired users (extra audio description).
    3. srclang → Language code (e.g., en, es).
    4. label → Name shown in the video player’s menu.

    The Robust & Professional Method (source)

    Why Use Instead of Just src?
    • Different browsers support different video formats.
    • (source) allows you to list multiple formats.
    • Browser will pick the first one it understands.

    Common Formats:
    1. .mp4 (H.264) → Works almost everywhere (best default).
    2. .webm (VP8/VP9) → Open-source, great for Chrome & Firefox.
    3. .ogg (Theora) → Older open-source option, less used today.

    How It Works (Step by Step):
    1. Browser checks the first → movie.mp4.
      • If it can play → it uses this and ignores the rest.
      • If not → it moves to the next source.
    2. Browser checks movie.ogg.
      • If it can play → it uses this.
      • If not → it looks for the next option.
    3. If none work → Shows the fallback message:
      • "Your browser does not support the video tag."
    Summary:
    • Pros: Maximum browser compatibility. Professional approach.
    • Cons: You need to create and host videos in multiple formats.

    Video Embedding Methods in HTML
    1. (source) Method (Professional)
      • Browser gets multiple options (mp4, webm, ogg).
      • It picks the first format it supports.
      • If none work → shows fallback message.
      2. Pros: Maximum compatibility, fallback support.
      Cons: Needs multiple files . src Method (Simple)
    2. Browser gets one direct file (e.g., a.mp4).
      • If it supports → plays.
      • If not → video fails (no fallback).
      3. Pros: Very simple, easy for quick use.
      Cons: No backup option, risky for public sites.
    Comparison Table
    Feature Example 1 (source Method) Example 2 (src Method)
    Browser Compatibility Excellent. Provides multiple formats as fallbacks. Good, but not guaranteed. Relies on a single format.
    Flexibility High. You can list many different formats. Low. Only one format is possible.
    Simplicity More complex. Requires multiple lines and files. Very simple. Requires only one line.
    Fallback for Old Browsers Excellent. Provides custom text if the <video> tag fails. None.
    Best Use Case Public-facing websites where you need to support all users. Quick tests, internal projects, or when you are 100% certain your target audience supports your one format.


    Conclusion: Which Video Method to Use?
    • For serious, public websites → Always use the method (best practice, supports all browsers).
    • For quick tests/internal projects → You can use the simple src method, but it’s a bit risky (no fallback).

    Note: Chromium browsers do not allow autoplay in most cases. However, muted autoplay is always allowed.
